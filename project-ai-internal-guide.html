PROJECT CASE STUDY

AI活用ナレッジサイト構築

点在していたAI活用ノウハウを誰もが使える「組織の資産」へ。インタラクティブな学習サイトの企画から構築までを主導したプロジェクト。

主要技術

目的

組織全体のAIリテラシー向上

背景と直面した困難

Situation: 点在するAIノウハウ

AIツールが導入されたものの、その活用は一部の詳しいメンバーに限られていました。ノウハウが組織内で十分に共有されておらず、同様の質問が繰り返される状況でした。組織全体としてAIを使いこなすための、体系化された知識ベースが存在しませんでした。

Difficulty: 「読まれない資料」の壁

当初作成したテキストベースの資料は、期待したほど活用されませんでした。「いつでも見られる」資料は「いつまでも見られない」という現実を痛感。どうすれば多忙な従業員が、自ら学びたいと思うような魅力的なコンテンツを提供できるか、という根本的な課題に直面しました。

チームでの課題解決プロセス

ニーズ分析とコンテンツ企画

まず、従業員がAIをどのような業務で使いたいのか、何に困っているのかをアンケートやヒアリングで徹底的に調査しました。その結果を基に、単なる機能説明ではなく、「目的別」「ツール別」に具体的な活用事例を学べるインタラクティブなガイド形式のサイトを企画しました。

AIとの協業によるコンテンツ制作

収集したユースケースを基に、生成AIを活用してコンテンツの草案を高速で生成。各ツールの専門知識を持つメンバーがその内容を監修・追記することで、質と量の両方を担保したコンテンツ制作フローを確立しました。

UI/UX設計とインタラクティブ機能の実装

「使っていて楽しい」学習体験を目指し、UI/UXを設計。AIに相談しながらHTML/CSS/JavaScriptのコードを生成し、目的やツールで事例を絞り込めるフィルター機能や、クリックすると詳細が表示されるアコーディオン機能などを実装。プロトタイプをチームで共有し、フィードバックを元に改善を重ねました。

展開とフィードバック収集

完成したナレッジサイトを組織のポータルで公開。利用状況を分析しつつ、定期的に利用者からのフィードバックを収集する仕組みを構築しました。新しい活用事例が生まれれば、それをサイトに追記していく「成長するナレッジベース」として運用を開始しました。

プロジェクトがもたらした価値

AIの日常的な活用

定着

AIが特別なツールではなく、日常業務の一部として浸透しました。

定期的なAI活用

高い水準

多くの従業員が定期的にAIを活用し、業務効率化ツールとして習慣化しました。

自発的な学習文化の醸成

「とりあえずサイトを見てみよう」という文化が浸透。従業員が能動的にスキルアップを目指す雰囲気が生まれました。

ナレッジの一元化

点在していたノウハウが体系的に集約され、誰もがアクセスできる「組織の資産」へと昇華しました。

成果物デモ

インタラクティブガイド

このプロジェクトの成果物として、実際に構築したインタラクティブガイドのデモを以下に示します。このガイドは、従業員が自身の目的に合ったAI活用法を直感的に見つけられるように設計されています。また、グローバルな利用を想定し、日本語と英語の切り替え機能を実装している点も特徴です。

学びと再現性

「静的な資料」から「動的な体験」へ

「読まれない資料」の壁に直面した当初の苦労は、大きな学びとなりました。情報はそこにあるだけでは価値を生まず、ユーザーが「自分ごと」として捉え、楽しみながら探求できる「体験」としてデザインすることが不可欠であると痛感しました。この経験は、単なるサイト構築に留まらず、組織内での知識共有のあり方そのものを見直すきっかけとなりました。

使用スキル

  • HTML5 (Semantic Markup)
  • Tailwind CSS
  • JavaScript (ES6+)
  • Generative AI (コンテンツ生成)
  • UI/UX Design Principles

再現ポイント

  • ユーザー中心設計: まず利用者のニーズを深く理解し、コンテンツを企画する。

  • AIとの協業: コンテンツ草案の生成にAIを活用し、専門家が監修することで制作を効率化する。

  • 継続的な改善: 公開後もフィードバックを収集し、コンテンツを更新し続ける仕組みを構築する。
