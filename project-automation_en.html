Reporting Process Automation

Background & Challenges

Situation: The Existing Reporting Process

Monthly reports were compiled manually by staff extracting data from the core system and manufacturing execution system, then aggregating it in spreadsheets. This process required significant time each month, creating a heavy workload. Additionally, there were concerns about data reliability due to potential manual errors like typos.

Difficulty: Tool Selection and Implementation

We initially considered automating with RPA alone, but found that the logic required to link different data formats and calculate new metrics would be complex to implement using standard functions alone. Combining appropriate tools and overcoming technical challenges was key to the project's success.

Problem-Solving Process

Step 1: Thorough Process Visualization

Through repeated interviews with staff, we mapped out the entire existing workflow. We created a flowchart detailing "what" was being reported, "from where," "how it was processed," and "to whom," identifying key points and challenges for automation.

Step 2: RPA Scenario Design

We designed and developed an RPA scenario with a clear role: to focus on the routine and reliable task of logging into each system and downloading the necessary CSV files.

Step 3: Development of Data Processing Logic with Python

To complement RPA's capabilities, we developed data processing logic using Python. This created a hybrid system where the script receives multiple CSVs from the RPA, then performs the merging, calculation, and formatting.

Step 4: Phased Rollout & Changed Information Sharing

The generated data was automatically linked to a BI dashboard on a shared folder. This allowed stakeholders to check the latest status in real-time, shifting the culture from "waiting" for reports to proactively "viewing" information when needed.

Takeaways & Replicability

Leveraging "Combination" Based on Tool Characteristics

We initially aimed for automation with RPA alone but faced challenges with complex data processing. This experience taught us the importance of a hybrid approach that leverages the strengths of different tools instead of sticking to just one. Combining RPA's "reliability in routine tasks" with Python's "flexible data processing capabilities" allowed us to achieve a higher level of automation than possible with a single tool. This insight has become valuable knowledge for advancing future DX projects.

Skills Used

  • Business Process Analysis

Replication Points

  • Hybrid Approach: Combine the strengths of multiple tools, like RPA and a programming language, to compensate for limitations.

  • Clear Role Division: Assign routine tasks like data collection to RPA and complex data processing to Python.

  • Process Decoupling: Separating the data processing logic improves maintainability, scalability, and makes it easier to consider reuse by other systems.
